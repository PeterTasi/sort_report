心得與結論

根據理論複雜度與上方模擬實驗結果，整理出以下觀察與建議。

O(n²) 的成長代價

Bubble、Selection、Insertion Sort 理論複雜度相同，但模擬結果顯示，當 n 從 1,000 增至 10,000，執行時間約增加 100 倍，與 O(n²) 預測完全吻合。大型資料集應避免使用此類演算法。

Quick Sort 實務優勢

Quick Sort 平均複雜度 O(n log n)，在隨機資料下表現最快，原因是常數因子小且快取命中率高。但最差情況（pivot 選擇不當）會退化到 O(n²)，因此 pivot 策略是關鍵。

Merge Sort 的穩定性

Merge Sort 無論資料分布如何，時間複雜度恆為 O(n log n)，並為穩定排序，適合需要保留相等元素順序的場景（如多欄位排序）。代價是需要 O(n) 額外空間。

Insertion Sort 的特殊情境

在小型資料（n < 20）或資料幾乎已排序的情況下，Insertion Sort 最佳複雜度為 O(n)，實際執行速度有時甚至優於 Quick Sort，因為其 overhead 極小。

綜合建議

一般用途首選 Quick Sort（速度快）或 Merge Sort（穩定可預期）。資料量小或接近有序時可考慮 Insertion Sort。Bubble Sort 與 Selection Sort 適合教學展示，不建議用於實際大型資料排序。
